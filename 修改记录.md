   - 输入 nav 文件或已解析 `nav_data`。
   - 输入 obs 文件或已解析 `obs_data`。
   - 输入固定经纬度高程，作为理论几何距离 `rho_gt_m` 的接收点。
   - 按 OBS 历元和卫星 ID 对齐，输出 `residual_tbl`。
   - 默认使用 GPS 单系统 `systems = {'G'}`，避免多星座系统间偏差混入单一接收机钟差。

   - `rho_gt_m`：根据 NAV、历元时间、固定经纬高，纯计算卫星到该点的理论几何距离。

   - 由每颗卫星给出一个 `clock_sample_m = P_i - rho_i + c * dt_s`。
   - 默认用同历元同系统所有可用卫星的均值作为 `receiver_clock_bias_m`。
   - 输出 `clock_tbl`，记录每个历元每个系统的接收机钟差、参与卫星数和钟差样本标准差。

4. 新增 `example_nav_obs_range_residuals.m`。
