# 基于多卫星距离残差的时空一致性认证

现有手势模板认证主要回答“恢复出的轨迹是否足够像预设手势模板”这一问题。该判据能够约束用户输入的动作形状，但仅凭轨迹相似性并不能充分保证该动作一定发生在指定时间和指定地点。原因在于，同一种手势，例如字母 `Z`，理论上可以在不同地点或不同时间被重复执行；只要恢复轨迹仍然接近模板，单纯的模板匹配分数就可能给出较高置信度。因此，StarDial 还需要引入一个独立的时空一致性判据，用来验证当前观测是否能够被指定的时间和地理位置共同解释。

GNSS 伪距观测天然包含时间和空间信息。对于第 `i` 颗卫星，在时间 `t`、接收机位置 `p` 下，原始伪距可以表示为

```text
P_i = rho_i(t, p) + c(delta t_r - delta t_i) + epsilon_i
```

其中，`rho_i(t, p)` 是卫星到接收机的真实几何距离，`delta t_r` 是接收机钟差，`delta t_i` 是卫星钟差，`c` 是光速，`epsilon_i` 包含观测噪声、多路径和剩余模型误差。利用导航星历可以计算卫星钟差并进行修正，得到

```text
P_i + c delta t_i = rho_i(t, p) + c delta t_r + epsilon_i
```

可以看到，修正后的伪距仍然包含一个接收机钟差项。该项对同一系统、同一历元内的所有卫星近似表现为公共偏移。因此，在验证某一个候选时空 `(t_c, p_c)` 时，StarDial 首先由导航星历计算每颗卫星到候选位置的理论几何距离

```text
rho_i^GT = rho_i(t_c, p_c)
```

然后利用所有可用卫星估计该候选时空下最优的接收机钟差公共项

```text
b_c = mean_i(P_i + c delta t_i - rho_i^GT)
```

最终得到每颗卫星的距离残差

```text
e_i = P_i + c delta t_i - b_c - rho_i^GT
```

并以多卫星残差的均方根作为时空一致性度量：

```text
S_ST(t_c, p_c) = sqrt(1/N * sum_i e_i^2)
```

当候选时空 `(t_c, p_c)` 与真实观测时空一致时，理论几何距离 `rho_i^GT` 与观测距离能够同时对齐；接收机钟差公共项被估计并扣除后，残差 `e_i` 主要由噪声、多路径和星历模型误差构成，因此 `S_ST` 应保持在较低水平。相反，当候选位置或候选时间错误时，理论距离与观测距离之间会产生多卫星相关的结构性偏差，使 `S_ST` 显著增大。

空间唯一性可以从距离的一阶近似中解释。若候选位置相对真实位置存在偏移 `Delta p`，则第 `i` 颗卫星的几何距离误差近似为

```text
Delta rho_i ≈ -u_i^T Delta p
```

其中 `u_i` 是从接收机指向第 `i` 颗卫星的视线单位向量。由于不同卫星分布在不同方向，它们对应的 `u_i` 并不相同，因此同一个位置偏移会在不同卫星上产生不同的距离变化。接收机钟差只能吸收所有卫星共同的公共偏移，无法消除这种由卫星方向差异造成的非公共残差。因此，只要候选位置偏离真实位置，多卫星残差向量就会出现发散，表现为 `S_ST` 增大。这说明距离残差能够在同一时间下区分不同地点。

时间唯一性同样来自多卫星几何距离随时间的差异性变化。即使接收机位置不变，卫星仍在持续高速运动，因此每颗卫星到接收机的距离随时间变化。若候选时间相对真实时间存在偏移 `Delta t`，则有近似关系

```text
Delta rho_i ≈ dot rho_i * Delta t
```

其中 `dot rho_i` 表示第 `i` 颗卫星相对于接收机的径向距离变化率。不同卫星的轨道位置、运动方向和视线投影不同，因此 `dot rho_i` 也不同。接收机钟差同样只能吸收公共时间偏移所表现出的共同距离项，无法同时补偿所有卫星各不相同的距离变化率。因此，当使用错误时间计算理论几何距离时，多卫星残差会随时间偏移迅速增大。这说明距离残差能够在同一地点下区分不同时间。

因此，StarDial 的时空一致性不是由单颗卫星的距离是否匹配来保证的，而是由多颗卫星组成的距离残差向量共同保证的。合法时空下，所有卫星的观测距离能够在扣除接收机钟差后同时与理论几何距离对齐；错误时空下，不同卫星产生的方向相关或时间相关距离误差无法被一个公共钟差项消除，从而形成可检测的残差模式。最终认证规则可以表示为

```text
Accept = GestureTemplatePass AND S_ST(t_c, p_c) < tau_ST
```

其中，`tau_ST` 是由合法样本或标定数据确定的时空一致性阈值。这样，手势模板认证用于保证“动作形状正确”，而距离域时空一致性认证用于保证“该动作确实发生在指定时间和指定地点”。二者相互独立又共同约束，构成从手势身份到物理时空的完整认证闭环。
